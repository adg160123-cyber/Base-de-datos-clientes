Eres experto en crear interfaces CLI intuitivas, robustas y altamente desacopladas, utilizando las últimas características de C# 14 y el SDK de .NET 10. <context>
Estamos finalizando una aplicación de consola en .NET 10 basada en Clean Architecture.

Estado del Proyecto:
- El sistema usa Inyección de Dependencias (Microsoft.Extensions.DependencyInjection).
- Los casos de uso de Clientes (Consultar todos, Consultar por ID, Registrar) ya están implementados.
- El punto de entrada es Program.cs (utilizando Top-level statements).
</context>

<task_description>
Debes modificar 'Program.cs' para implementar un menú interactivo y profesional que permita al usuario ejecutar las acciones de gestión de clientes. El sistema debe ser resiliente a errores de entrada y seguir el principio de Inversión de Dependencias.
</task_description>

<technical_requirements>

<rule_set name="Interfaz y Menú">

Implementar un menú infinito (while loop) con las opciones:

1. Consultar todos los clientes
2. Consultar cliente por ID (Validar: solo enteros)
3. Registrar nuevo cliente
4. Salir

El registro de clientes debe solicitar cada propiedad individualmente
(nombre, correo, teléfono, etc.) con validación inmediata.

</rule_set>

<rule_set name="C# 14 y Limpieza">

- Utilizar Raw String Literals (triple comillas) para el diseño del menú.
- Inyectar los Casos de Uso mediante un IServiceScope manual.
- No resolver servicios Scoped desde el root provider.
- Si la lógica de registro es extensa, crear un método local para mantener Program.cs limpio.

</rule_set>

<rule_set name="Configuración de Ejecución y Secretos">

- Configurar el archivo Properties/launchSettings.json para establecer DOTNET_ENVIRONMENT como Development.
- Asegurar que los User Secrets se carguen si el entorno es Development.
- Validar que la cadena de conexión no esté hardcodeada.

</rule_set>

<rule_set name="Verificación de Entorno">

Confirmar la existencia de la carpeta Properties/ y el archivo launchSettings.json.
Si no existen, crearlos con el perfil de ejecución adecuado.

</rule_set>

</technical_requirements>

<execution_workflow>

1. Explorar src/Application/ y src/Core/UseCases/ para identificar los casos de uso de Client.
2. Configurar launchSettings.json para habilitar entorno Development.
3. Verificar configuración de ServiceCollection en Program.cs.
4. Diseñar flujo de captura de datos para registrar un nuevo cliente.
5. Implementar menú interactivo con llamadas a los UseCases.
6. Validar compilación con dotnet build y ejecución con dotnet run.

</execution_workflow>

<output_format>

- Código completo y actualizado de Program.cs.
- Contenido del archivo Properties/launchSettings.json.
- Código de archivos de soporte si se crean.
- Sección <environment_report> confirmando el entorno de ejecución.
- Breve guía de uso del menú generado.

</output_format>
