<technical_requirements>

<rule_set name="Acceso a Datos">

- Utilizar exclusivamente ADO.NET (SqlConnection, SqlCommand, SqlDataReader).
- Evitar completamente Dapper, Entity Framework o cualquier ORM dinámico.
- Todas las consultas SQL deben ser explícitas y optimizadas.
- La conexión debe obtenerse mediante IDbConnectionFactory.
- Realizar cast a SqlConnection para usar métodos asíncronos nativos.

</rule_set>

<rule_set name="Operaciones del Repositorio">

El repositorio debe implementar las siguientes operaciones:

- GetByIdAsync
- GetAllAsync
- AddAsync
- UpdateAsync
- DeleteAsync

Las consultas deben trabajar contra la tabla:

Clients

</rule_set>

<rule_set name="Características de C# 14">

- Utilizar Primary Constructors para inyección de dependencias.
- Usar `field` cuando sea necesario para validaciones de estado.
- Implementar `IAsyncEnumerable<Client>` en `GetAllAsync` usando `yield return`.
- Mantener el código completamente compatible con Native AOT.

</rule_set>

</technical_requirements>
