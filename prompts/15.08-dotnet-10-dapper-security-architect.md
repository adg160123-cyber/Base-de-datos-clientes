<task_workflow>

<step id="1">

<description>
Configuración de Secretos de Desarrollo
</description>

<action>
Ejecuta:

dotnet user-secrets init

Extrae el 'user id' y el 'pwd' de la cadena de conexión proporcionada.

Almacena la cadena segura utilizando:

dotnet user-secrets set "ConnectionStrings:DefaultConnection" ""

La cadena original en appsettings.json debe quedar sin credenciales.
</action>

</step>

<step id="2">

<description>
Gestión de Archivos de Configuración
</description>

<action>

Crear los archivos:

appsettings.json  
appsettings.Development.json  
appsettings.Production.json

Asegurar que el entorno Development cargue automáticamente los User Secrets.

</action>

</step>

<step id="3">

<description>
Arquitectura de Persistencia SOLID
</description>

<action>

Implementar:

IDbConnectionFactory  
SqlConnectionFactory

Responsabilidad:

- Gestionar el ciclo de vida de IDbConnection.
- Servir conexiones a los repositorios como ClientRepositoryImpl.

Usar la palabra clave 'field' en las propiedades para validar que la cadena de conexión sea válida al asignarse.

</action>

</step>

<step id="4">

<description>
Registro de Dependencias con Extension Blocks
</description>

<action>

Crear un bloque:

extension(IServiceCollection services)

para registrar:

IDbConnectionFactory  
SqlConnectionFactory

Configurar Program.cs usando:

Host.CreateApplicationBuilder(args)

</action>

</step>
